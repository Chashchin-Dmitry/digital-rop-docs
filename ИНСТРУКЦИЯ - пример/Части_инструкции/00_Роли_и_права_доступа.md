# Роли и права доступа

## Типы ролей в системе

В системе "Цифровой РОП" существует **4 типа ролей** с различными уровнями доступа:

1. **Супер-администратор**
2. **Администратор** 
3. **Руководитель офиса**
4. **Менеджер**

## Права супер-администратора

### Полный доступ ко всем функциям

**Просмотр данных**:
- Все звонки всех менеджеров
- Статистика по всем офисам и подразделениям
- Полная аналитика и отчеты

**Управление системой**:
- Настройка API ключей и интеграций
- Создание и редактирование чек-листов
- Управление промтами для ИИ-анализа
- Настройка таблиц и полей аналитики

**Управление пользователями**:
- Назначение ролей другим пользователям
- Создание администраторов
- Управление доступом к офисам

**Уникальные возможности**:
- Доступ к разделу "Слова-паразиты"
- Настройка технических параметров подключения
- Просмотр статистики обработки файлов

## Права администратора

### Расширенный доступ с ограничениями

**Аналитика и данные**:
- Просмотр звонков (объем доступа может быть ограничен)
- Статистика по менеджерам и офисам
- Генерация отчетов и аналитики

**Настройки системы**:
- Создание и редактирование чек-листов
- Настройка промтов для таблиц
- Управление конверсиями и тегами
- Создание кастомных аналитических таблиц

**Управление тестами**:
- Редактирование автоматически сгенерированных тестов
- Утверждение тестов для отправки менеджерам
- Контроль процесса тестирования

**Технические функции**:
- Настройка API ключей
- Управление веб-хуками
- Доступ к чату с ИИ

**Просмотр звонков**:
- **Только собственные** звонки и их анализ
- Транскрипции разговоров
- Оценки по каждому этапу скрипта
- Рекомендации по улучшению

**Коммуникации. Общий вид.png**

**Прохождение тестов**:
- Просмотр тестов, утвержденных руководителем
- Прохождение тестов в системе
- Просмотр результатов тестирования

**Тесты. пример теста.png**

**Аналитика**:
- Собственная статистика производительности
- Динамика улучшения показателей
- История прохождения тестов

**Строгие ограничения**:
- Нет доступа к данным других менеджеров
- Не может видеть настройки системы
- Не может редактировать тесты
- Нет доступа к административным функциям

## Управление правами доступа

### Назначение ролей

**Пользователи. Выбор прав.png**

**Кто может назначать роли**:
- **Супер-администратор** - может назначить любую роль
- **Администратор** - может назначать роли менеджеров и руководителей

**Процесс назначения**:
1. Переход в раздел "Настройки → Пользователи"
2. Поиск нужного сотрудника
3. Выбор соответствующей роли
4. Сохранение изменений

### Интеграция с LDAP

**Автоматическая синхронизация**:
- Пользователи используют учетные данные из CRM Setl Group
- Автоматическое создание аккаунтов при первом входе
- Синхронизация с корпоративной структурой

**Преимущества LDAP-интеграции**:
- Единая система аутентификации
- Нет необходимости в отдельной регистрации
- Автоматическое отключение при увольнении сотрудника

## Ограничения по данным

### Территориальные ограничения

**По офисам**:
- Руководители видят только данные своего офиса
- Менеджеры видят только собственные данные
- Администраторы могут иметь доступ ко всем данным

### Временные ограничения

**Фильтрация по периодам**:
- Все роли могут использовать фильтры по датам
- Объем доступных исторических данных может различаться
- Возможность экспорта данных зависит от роли

### Функциональные ограничения

**Редактирование**:
- Только администраторы и руководители могут редактировать тесты
- Настройки доступны только администраторам
- Менеджеры не могут изменять чек-листы или промты

**Важно**: Система автоматически скрывает недоступные функции для каждой роли, что обеспечивает безопасность и удобство использования.
