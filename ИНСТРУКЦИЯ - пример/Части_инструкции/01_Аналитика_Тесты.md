# 1. Аналитика - Тесты

Раздел **Тесты** обеспечивает автоматическое обучение менеджеров на основе их ошибок в звонках. Система анализирует невыполненные этапы [чек-листов](#настройки-скрипты-и-промты) и генерирует персонализированные тесты для повышения качества работы.

## Общий вид раздела

**Тесты. Общий вид.png** - Основной интерфейс управления тестами

{INTERFACE} Раздел предоставляет выбор периода для просмотра тестов, фильтрацию по менеджерам и статусу теста, список тестов с возможностью раскрытия детальной информации, а также кнопки управления для редактирования и одобрения тестов руководителем.

### Статусы тестов:
- **Не одобрен** (синяя кнопка) - тест создан системой, ожидает одобрения руководителя
- **Не пройден** (красная кнопка) - тест одобрен руководителем и отправлен менеджеру  
- **Пройден** - тест выполнен менеджером

## Как формируются тесты

{TECHNICAL} Алгоритм автоматического создания тестов: система анализирует ошибки менеджера в звонке и выбирает ОДНУ ошибку (даже если было несколько). Формирует тест на основе настроенного шаблона скрипта: вопрос создается на базе названия невыполненного этапа чек-листа, правильный ответ берется из описания "когда считаем этап выполненным" из настроек этапа, неправильные ответы (3 варианта) генерируются ИИ. Готовый тест попадает к руководителю на одобрение.

## Процесс одобрения

{INTERFACE} Важно: тесты НЕ отображаются у менеджера до одобрения руководителем! Workflow одобрения включает создание теста со статусом "Не одобрен", проверку корректности формулировок руководителем, редактирование при необходимости, изменение статуса на "Не пройден" после одобрения. Только после этого тест появляется в интерфейсе менеджера.

## Пример теста

**Тесты. пример теста.png** - Детальный вид раскрытого теста

При раскрытии теста отображается:
- **Вопрос** - сформирован на основе этапа чек-листа
- **Варианты ответов** - строго 4 варианта (1 правильный + 3 от ИИ)
- **Статус ответа**: ❌ неправильный / ✅ правильный ответ
- **Ответ менеджера** - выбор сотрудника при прохождении

## Управление тестами

**Тесты. 1 - редактирование теста 2 - одобрить тест для отправки в аккаунт менеджеру.png** - Кнопки управления тестами

{INTERFACE} Руководитель может редактировать тест (кнопка с карандашом) для изменения формулировок вопроса и ответов, а также одобрить тест (зеленая галочка) для отправки менеджеру. После одобрения тест становится доступен сотруднику для прохождения.

## Редактирование теста

**Тесты. Пример редактирование теста.png** - Интерфейс редактирования теста

{INTERFACE} В режиме редактирования доступно изменение формулировки вопроса и редактирование всех 4-х вариантов ответов. Важное ограничение: всегда строго 4 варианта (нельзя добавить или удалить). После внесения изменений можно отменить или обновить данные теста.
