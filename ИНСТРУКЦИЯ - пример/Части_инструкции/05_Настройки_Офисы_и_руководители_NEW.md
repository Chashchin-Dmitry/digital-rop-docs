# 5. Настройки - Офисы и руководители

Раздел для управления структурой офисов и назначения руководителей в системе.

## Общий вид

**Офисы и руководители. Общий вид.png**

{INTERFACE} Таблица отображает структуру организации с колонками: номер офиса (#), название офиса/подразделения и назначенные руководители. Система показывает как офисы с одним руководителем ("ПСО → Елена Красильникова"), так и с несколькими руководителями ("Дилерские продажи → Наталья Сабаева, Елена Гуторова, Анна Сергеенкова").

## Принцип работы структуры

### Типы назначений руководителей

{INTERFACE} Система поддерживает три варианта:
- **Пустые ячейки** - офисы без назначенных руководителей (данные еще не настроены)
- **Одиночные назначения** - один руководитель отвечает за весь офис
- **Множественные назначения** - несколько руководителей совместно управляют офисом

### Автоматическая синхронизация

{TECHNICAL} Информация об офисах и руководителях синхронизируется автоматически из CRM Setl Group. Система использует числовые ID офисов для внутренней работы, а сотрудники идентифицируются через корпоративные UUID. Обновление происходит регулярно без участия администратора.

## Связь с аналитическими разделами

### Фильтрация данных

{INTERFACE} Офисы из этого списка автоматически появляются в фильтрах всех аналитических разделов как опция "Поиск по офису". Это позволяет руководителям быстро находить данные по своим подразделениям, а администраторам - анализировать работу конкретных офисов.

### Система доступа

{TECHNICAL} Руководители видят данные только по своим офисам согласно настроенным правам. Менеджеры имеют доступ только к собственной статистике. Суперадминистратор видит информацию по всем офисам без ограничений.

## LDAP интеграция и роли

### Корпоративная аутентификация

{TECHNICAL} Система интегрирована с Active Directory Setl Group для аутентификации пользователей. Роли определяются через группы в AD, что обеспечивает централизованное управление доступом без дублирования учетных записей.

### Особенности привязки к офисам

{INTERFACE} В корпоративной системе нет жесткой привязки сотрудников к офисам (используется только поле "адрес"). Это учитывает мобильность современных сотрудников, которые могут работать удаленно или в разных офисах. Права доступа настраиваются через специальные группы в Active Directory.

## Уровни доступа в системе

### Администраторы (Суперадминистратор)
{INTERFACE} Полный доступ ко всем офисам, возможность просмотра любых данных, управление настройками системы, контроль работы всех подразделений.

### Руководители офисов
{INTERFACE} Доступ только к данным своих подразделений, утверждение тестов для подчиненных, просмотр аналитики по своей команде, настройка локальных параметров офиса.

### Менеджеры
{INTERFACE} Доступ только к собственным данным, прохождение тестов, просмотр личной статистики, работа с назначенными заданиями.

## Управление структурой

### Гибкость организации

{INTERFACE} Система поддерживает сложные организационные структуры с множественными руководителями, матричную подчиненность, временные назначения и перекрестное управление проектами.

{TECHNICAL} Структура обновляется автоматически при изменениях в CRM, что обеспечивает актуальность данных и исключает необходимость ручной синхронизации. Система адаптируется к реорганизациям и структурным изменениям в компании.

## Практическое применение

### Для руководителей
- Быстрый доступ к данным своей команды
- Фильтрация отчетов по подразделению
- Контроль выполнения задач менеджерами
- Утверждение обучающих тестов

### Для администраторов
- Контроль работы всех офисов
- Сравнительный анализ эффективности
- Управление общими настройками
- Мониторинг системной активности
