# 5. Настройки - Пользователи

Раздел для управления пользователями системы и их ролями доступа.

## Общий вид

**Пользователи. общий вид.png**

{INTERFACE} Основная страница отображает таблицу всех пользователей системы с колонками: ID, ФИО, роль, email для входа и статус активности (ВКЛ/ВЫКЛ). Каждая строка содержит кнопки "Изменить" и "Удалить" для управления учетными записями.

{TECHNICAL} Система поддерживает три основные роли с различными уровнями доступа:
- **Администратор** - полный доступ ко всем функциям и данным
- **Руководитель офиса** - управление своим офисом и утверждение тестов
- **Менеджер** - доступ только к собственным данным

## Поиск пользователей

**Пользователи. Работа поиска пользователей.png**

{INTERFACE} Поле поиска в верхней части страницы позволяет фильтровать список пользователей. Поиск работает в режиме реального времени по всем полям: имя, фамилия, отчество и email. Результаты отображаются мгновенно при вводе текста.

## Редактирование пользователя

**Пользователи. Редактирование пользвоателя на кнопку изменить.png**

При нажатии кнопки "Изменить" открывается форма редактирования учетной записи.

{INTERFACE} Форма содержит следующие поля:
- **ФИО** - полное имя пользователя
- **Email** - корпоративный адрес для входа в систему
- **Роль в системе** - выпадающий список с доступными ролями
- **Пароль** - необязательное поле для смены пароля
- **Статус** - переключатель ВКЛ/ВЫКЛ для активации/деактивации

После внесения изменений нажмите кнопку "Сохранить" для применения настроек.

## Выбор ролей в системе

**Пользователи. Выбор прав.png**

{TECHNICAL} Система ролей определяет уровень доступа каждого пользователя:

### Менеджер
- Просмотр только собственных звонков и аналитики
- Прохождение назначенных тестов
- Доступ к личной статистике

### Руководитель офиса
- Полный доступ к данным своего офиса
- Утверждение тестов для менеджеров
- Просмотр аналитики по подчиненным
- Экспорт отчетов по офису

### Администратор
- Полный доступ ко всем данным системы
- Настройка чек-листов и промтов
- Управление пользователями и структурой
- Техническое администрирование

## Интеграция с LDAP

{TECHNICAL} Система использует LDAP-аутентификацию для интеграции с корпоративной инфраструктурой Setl Group. Пользователи входят в систему с корпоративными учетными данными без необходимости создания отдельных паролей.

{INTERFACE} Активные пользователи имеют заполненное поле email и статус "ВКЛ". Неактивные пользователи отображаются со статусом "ВЫКЛ" и могут иметь пустое поле email до активации.

## Управление доступом

{INTERFACE} Администраторы могут:
- Добавлять новых пользователей через кнопку "Добавить пользователя"
- Изменять роли существующих пользователей
- Временно отключать доступ через статус "ВЫКЛ"
- Удалять неиспользуемые учетные записи

{TECHNICAL} При изменении роли пользователя его права доступа обновляются немедленно. Это позволяет оперативно управлять уровнем доступа без перезагрузки системы.
