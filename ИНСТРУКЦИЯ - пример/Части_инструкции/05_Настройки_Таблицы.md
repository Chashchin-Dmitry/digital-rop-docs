# 5. Настройки - Таблицы

Раздел для создания пользовательских аналитических таблиц с настраиваемыми полями и ИИ-промтами для анализа звонков.

## Общий вид

**Таблицы. Общий вид.png**

Главная страница управления аналитическими таблицами.

### Список существующих таблиц:
- **"Недопустимые действия"** - готовая таблица для анализа
- **"Общие"** - дополнительная категория таблиц

### Функции управления:
- **"Добавить таблицу"** - создание новой аналитической таблицы
- **"Изменить таблицу"** - редактирование параметров существующей таблицы
- **"Изменить поля"** - настройка полей и промтов для таблицы
- **"Удалить"** - удаление неиспользуемой таблицы

## Создание новой таблицы

**Таблицы. Добавить таблицу.png**

Форма создания пользовательской аналитической таблицы.

### Поля для заполнения:
- **"Название"** - наименование таблицы (например: "Общая аналитика звонков")
- **"Категория"** - выпадающий список категорий (например: "Общие")
- **"Воронка"** - выпадающий список воронок продаж (например: "Новые")
- **"Урл для отправки результата"** - webhook URL для получения данных аналитики
  - Пример: `http://testurl.com/analytics_results`

### Кнопки управления:
- **"Отмена"** - отмена создания таблицы
- **"Сохранить"** - создание таблицы с указанными параметрами

## Настройка полей таблицы

**Таблицы. Настроенные поля - общий вид.png**

Детальная настройка полей для таблицы "Недопустимые действия".

### Структура настроенных полей:

#### Поле 1: "Менеджер использует"
- **Тип поля**: "Текст (ответ до 1000 букв)"
- **Задача**: "Необходимо найти нецензурную лексику в словах менеджера"
- **Что искать**: (детальный промт для ИИ)

#### Поле 2: "Менеджер демонстрат"
- **Тип поля**: "Текст (ответ до 1000 букв)"
- **Задача**: "МРК демонстративно вешает трубку"
- **Что искать**: "[Резкое прекращение разговора]"

#### Поле 3: "Менеджер спорит с кл"
- **Тип поля**: "Текст (ответ до 1000 букв)"
- **Задача**: "Разговор происходит на повышенных тонах. МРК спорит с клиентом"
- **Что искать**: (промт для определения конфликта)

### Функции управления полями:
- **"Добавить поле"** - создание нового поля анализа
- **"Изменить"** - редактирование существующего поля
- **"Удалить"** - удаление неиспользуемого поля

## Создание поля таблицы

### Шаг 1. Название поля

**Таблицы. Добавить поле. 1 - название поля.png**

Первый шаг - указание названия поля.

### Поля:
- **"Название"** - наименование поля (например: "Менеджер неуверенно представлял продукт")
- **Кнопка "Вперед"** - переход к следующему шагу

### Шаг 2. Тип данных

**Таблицы. Добавить поле. 2 - выбрать тип поля (число текст дата и тд).png**

Выбор типа данных для поля.

### Доступные типы:
- **"Текст (ответ до 1000 букв)"** - текстовое поле с ограничением (выбрано в примере)
- **Другие типы**: число, дата, логические значения

### Кнопки:
- **"Назад"** - возврат к предыдущему шагу
- **"Вперед"** - продолжение настройки

### Шаг 3. ИИ-промт

**Таблицы. Добавить поле. 3 - написать промт для ии под поле.png**

Настройка промта для ИИ-анализа.

### Настройка промта:
- **"Текст Промта"** - большое текстовое поле для детального описания
- **Пример промта**:
  ```
  Изучить, уверенно ли презентовал продукт клиенту 
  наш менеджер!
  
  1 - если уверено презентовал продукт (услуга, 
  покупка ЖК и тд.)
  0 - если не уверенно презентовал продукт (услуга, 
  покупка ЖК и тд.)
  
  Формат ответа
  1 или 0
  ```

### Кнопки:
- **"Назад"** - возврат к предыдущему шагу
- **"Сохранить"** - создание поля с промтом

## Принцип работы аналитических таблиц

### Анализ через ИИ (Webhook `analytics`):
1. **ИИ анализирует аудио** по настроенным промтам каждого поля
2. **Обработка промтов**: каждое поле получает результат согласно своему промту
3. **Формат ответов**: зависит от типа поля (текст, число, логическое значение)
4. **Отправка результата**: данные отправляются через webhook на URL таблицы

### Гибкость настройки:
- **Любые промты** - пользователь сам определяет что анализировать
- **Разные типы данных** - текст, числа, даты для различных задач
- **Индивидуальные URL** - каждая таблица может иметь свой endpoint
- **Категоризация** - группировка таблиц по категориям и воронкам

## Связь с другими разделами

### Использование результатов:
- **Раздел "Таблицы" (основной)** - отображение готовых аналитических данных
- **Фильтрация данных** - анализ по офисам, менеджерам, периодам

### Техническая интеграция:
- **Webhook отправка** - результаты таблиц передаются через индивидуальные URL
- **API ключ** - аутентификация для безопасной передачи данных аналитики
- **Структурированные данные** - единая структура для всех таблиц с переменным количеством полей

## Примеры использования

### Контроль качества:
- **"Недопустимые действия"** - отслеживание нецензурной лексики, конфликтов
- **"Профессионализм"** - анализ уверенности презентации, знания продукта
- **"Эмоциональное состояние"** - тон разговора, стрессовые ситуации

### Бизнес-аналитика:
- **"Конверсии"** - отслеживание целевых действий клиентов
- **"Возражения"** - анализ частых возражений и способов их обработки
- **"Потребности клиентов"** - выявление ключевых запросов и интересов

## Ключевые особенности:

1. **Полная настраиваемость** - пользователь создает любые поля с любыми промтами
2. **ИИ-анализ по промтам** - система анализирует звонки согласно заданным критериям
3. **Разные типы данных** - поддержка текста, чисел, дат для различных задач анализа
4. **Индивидуальные webhook-и** - каждая таблица может иметь свой URL для результатов
5. **Категоризация и воронки** - организация таблиц по бизнес-процессам
6. **Гибкая структура** - количество полей определяется потребностями пользователя
7. **Интеграция с CRM** - автоматическая отправка результатов во внешние системы
