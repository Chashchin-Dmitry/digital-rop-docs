# 5. Настройки - Таблицы

Раздел для создания пользовательских аналитических таблиц с настраиваемыми полями и ИИ-анализом.

## Общий вид

**Таблицы. Общий вид.png**

{INTERFACE} Страница отображает список созданных аналитических таблиц. Каждая таблица показывает название, категорию, количество настроенных полей и кнопки управления "Изменить" и "Удалить". Кнопка "Добавить таблицу" позволяет создать новую таблицу.

### Примеры таблиц

- **"Недопустимые действия"** - таблица для отслеживания нарушений в разговоре
- **"Продажа"** - анализ продающих действий менеджера  
- **"Test"** - тестовая таблица для проверки функций

## Создание новой таблицы

**Таблицы. Добавить таблицу.png**

{INTERFACE} Форма создания содержит поля для названия таблицы, выбора категории из выпадающего списка, выбора воронки и URL для отправки результатов. После создания основной структуры можно переходить к добавлению полей.

### Основные параметры

- **Название таблицы** - краткое описательное название
- **Категория** - группировка таблиц по назначению
- **Воронка** - привязка к этапам продаж
- **URL результатов** - адрес для получения данных анализа

## Настройка полей таблицы

**Таблицы. Настроенные поля - общий вид.png**

{INTERFACE} После создания таблицы отображается список настроенных полей с их типами данных и промтами для ИИ-анализа. Каждое поле можно редактировать или удалить. Кнопка "Добавить поле" позволяет расширить структуру таблицы.

## Пошаговое добавление поля

### Шаг 1. Название поля

**Таблицы. Добавить поле. 1 - название поля.png**

{INTERFACE} Указывается название поля, которое будет отображаться в аналитической таблице. Название должно четко отражать тип информации, которая будет извлекаться из звонков.

### Шаг 2. Тип данных

**Таблицы. Добавить поле. 2 - выбрать тип поля (число текст дата и тд).png**

{INTERFACE} Выбор типа данных для поля из выпадающего списка: "Текст (ответ до 1000 букв)", числовые значения, даты и другие форматы. Тип определяет как система будет обрабатывать и отображать результаты анализа.

### Шаг 3. ИИ-промт

**Таблицы. Добавить поле. 3 - написать промт для ии под поле.png**

{INTERFACE} Настройка промта для ИИ-анализа в большом текстовом поле. Промт должен четко описывать что именно нужно найти и извлечь из звонка, в каком формате предоставить ответ.

{TECHNICAL} Пример промта: "Изучить, уверенно ли презентовал продукт клиенту наш менеджер! 1 - если уверенно презентовал продукт, 0 - если не уверенно презентовал продукт. Формат ответа: 1 или 0"

## Принцип работы аналитических таблиц

{TECHNICAL} После создания таблицы с полями ИИ анализирует каждый звонок согласно настроенным промтам. Система извлекает нужную информацию и структурирует данные для просмотра в разделе [Таблицы](#таблицы). Результаты также отправляются на указанный URL для интеграции с внешними системами.

## Гибкость настройки

{INTERFACE} Администратор полностью контролирует структуру аналитики:
- Любые промты для извлечения нужной информации
- Разные типы данных для различных задач анализа  
- Группировка таблиц по категориям и бизнес-процессам
- Настраиваемое количество полей под конкретные потребности

## Примеры использования

### Контроль качества
- **"Недопустимые действия"** - отслеживание нецензурной лексики, конфликтов
- **"Профессионализм"** - анализ уверенности презентации, знания продукта

### Бизнес-аналитика  
- **"Конверсии"** - отслеживание целевых действий клиентов
- **"Возражения"** - анализ частых возражений и способов их обработки
- **"Потребности клиентов"** - выявление ключевых запросов и интересов
