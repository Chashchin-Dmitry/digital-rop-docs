# Администрирование системы

Этот раздел описывает основные задачи администратора по управлению и настройке системы "Цифровой РОП".

## Обзор административных функций

{INTERFACE} Администратор имеет доступ ко всем разделам системы и отвечает за техническую настройку, управление пользователями, настройку критериев оценки и мониторинг работы системы. Все административные функции доступны через главное меню в разделе "Настройки".

## Управление пользователями

**Пользователи. общий вид.png**

Основная задача администратора - управление учетными записями и правами доступа в разделе ["Настройки → Пользователи"](#настройки-пользователи).

{INTERFACE} Таблица пользователей отображает всех сотрудников системы с указанием их ID, ФИО, назначенной роли, email для входа и статуса активности. Администратор может добавлять новых пользователей, редактировать существующие записи и управлять статусом активности.

### Роли и права доступа

**Пользователи. Выбор прав.png**

{TECHNICAL} В системе предусмотрены три основные роли:
- **Менеджер** - доступ только к собственным звонкам и тестам
- **Руководитель офиса** - доступ к данным всего офиса, утверждение тестов
- **Администратор** - полный доступ ко всем функциям и данным системы

### Интеграция с корпоративной системой

{TECHNICAL} Аутентификация пользователей происходит через LDAP-интеграцию с корпоративной системой Setl Group. Это означает, что сотрудники используют свои корпоративные логины и пароли для входа в систему без необходимости создания отдельных учетных записей.

## Настройка структуры организации

**Офисы и руководители. Офисы и руководители без редактирования.png**

Управление организационной структурой осуществляется в разделе ["Настройки → Офисы и руководители"](#настройки-офисы-и-руководители).

{INTERFACE} Интерфейс отображает иерархическую структуру компании с офисами, руководителями и менеджерами. Администратор может создавать новые офисы, назначать руководителей и распределять менеджеров по подразделениям.

## Настройка критериев оценки

**Скрипты и промты. Пример настроенного шаблона скрипта.png**

Ключевая функция администратора - настройка чек-листов для оценки звонков в разделе ["Настройки → Скрипты и промты"](#скрипты-и-промты).

{INTERFACE} Система позволяет создавать многоэтапные чек-листы с детальными критериями оценки. Для каждого этапа настраивается название, промт для искусственного интеллекта и параметры оценки. Администратор может создавать разные чек-листы для различных типов звонков.

{TECHNICAL} Качество промтов напрямую влияет на точность оценки. Промты должны быть написаны четко и однозначно, чтобы ИИ мог корректно определить выполнение каждого этапа. Рекомендуется тестировать новые промты на контрольной выборке звонков.

## Настройка аналитических таблиц

**Таблицы. Общий вид.png**

Создание пользовательских аналитических отчетов выполняется в разделе ["Настройки → Таблицы"](#настройки-таблицы).

{INTERFACE} Администратор может создавать новые таблицы с произвольным набором полей для извлечения специфической информации из звонков. Для каждого поля настраивается промт, определяющий, какие данные должен извлечь искусственный интеллект.

## Техническая интеграция

**Подключение. Технические параметры для отправки аудио в систему.png**

Настройка интеграции с внешними системами происходит в разделе ["Настройки → Подключение"](#настройки-подключение).

{TECHNICAL} Страница содержит:
- **API ключ** для безопасной передачи данных в систему
- **Webhooks** для отправки результатов анализа
- **Технические параметры** для настройки интеграции с CRM

Все параметры должны быть точно скопированы в настройки интегрируемой системы.

## Мониторинг работы системы

**Статистика файлов. Общий вид.png**

Контроль технического состояния осуществляется через раздел ["Другое → Статистика файлов"](#статистика-файлов).

{INTERFACE} Страница отображает:
- Количество загруженных аудиофайлов
- Количество обработанных записей
- Файлы в очереди на обработку
- Ошибки обработки

Регулярный мониторинг позволяет оперативно выявлять и устранять технические проблемы.

## Управление качеством речи

**Слова-паразиты. Общий вид.png**

Настройка контроля качества речи выполняется в разделе ["Дополнительные настройки → Слова-паразиты"](#слова-паразиты).

{INTERFACE} Администратор управляет словарем нежелательных слов и выражений, которые автоматически выделяются в транскриптах звонков. Это помогает менеджерам контролировать качество своей речи.

## Рекомендации по администрированию

### Регулярные задачи

{INTERFACE} Администратор должен регулярно:
- Проверять статистику обработки файлов
- Анализировать эффективность промтов
- Обновлять словарь слов-паразитов
- Контролировать корректность организационной структуры

### Оптимизация системы

{TECHNICAL} Для повышения качества работы системы рекомендуется:
- Собирать обратную связь от пользователей
- Тестировать новые промты перед внедрением
- Регулярно анализировать спорные оценки
- Документировать все изменения в настройках
