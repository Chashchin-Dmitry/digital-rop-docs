# Полный цикл обработки звонка

## Этап 1: Получение аудиофайлов

**Подключение. Технические параметры для отправки аудио в систему.png** - Настройки API для получения звонков

{TECHNICAL} CRM Setl Group автоматически отправляет аудиофайлы в систему через API после завершения звонка. Передается URL аудиофайла и метаданные: ID менеджера, ID звонка, ID сделки, ID стадии сделки. Система скачивает аудиофайл по URL и начинает обработку.

## Этап 2: Транскрибация аудио

{INTERFACE} Система автоматически конвертирует речь в текст с временными метками. Обработка занимает 2-5 минут в зависимости от длительности звонка.

## Этап 3: Разделение спикеров

{TECHNICAL} Система автоматически определяет участников разговора и помечает их как "Спикер 0" (менеджер) и "Спикер 1" (клиент). Каждая реплика привязывается к временной метке.

## Этап 4: Контроль качества

{TECHNICAL} При обнаружении нецензурной лексики запускается повторная обработка для исключения ложных срабатываний из-за плохого качества связи. Система сохраняет наиболее точную транскрипцию.

## Этап 5: Анализ по чек-листам

{INTERFACE} ИИ получает транскрипцию и настроенные [чек-листы](#настройки-скрипты-и-промты), анализирует выполнение каждого этапа скрипта и присваивает оценки: +1 (выполнено) или -1 (не выполнено). Рассчитывает процентное выполнение каждого этапа согласно настроенной строгости оценки.

{TECHNICAL} Система анализирует соблюдение установленных этапов разговора, качество выполнения каждого пункта и соответствие корпоративным стандартам. ИИ создает комментарии с объяснением каждой оценки, конкретные рекомендации по улучшению и точные процентные оценки качества выполнения.

## Этап 6: Кастомная аналитика по таблицам

{INTERFACE} Система последовательно проходит по каждой созданной [аналитической таблице](#настройки-таблицы), выполняет промты для каждого поля таблицы, заполняет результаты аналитики и сохраняет данные для отчетов.

Типы анализа включают извлечение специфической информации, анализ эмоций и тональности, определение ключевых моментов разговора согласно настроенным промтам.

## Этап 7: Анализ конверсий

{TECHNICAL} При настроенных конверсиях ИИ анализирует звонок по каждому промту конверсии, определяет достигнутые результаты, фиксирует успешные конверсии и рассчитывает показатели эффективности.

Возможные конверсии: запись на встречу, получение контактов, заключение сделки и другие настраиваемые цели компании.

## Этап 8: Генерация тестов для обучения

{INTERFACE} При наличии этапов с оценкой -1 балл система автоматически создает **один тест на звонок** на основе наиболее критичных ошибок. Тест формируется для конкретного менеджера и требует одобрения руководителя перед отправкой.

{TECHNICAL} ИИ генерирует вопросы на основе допущенных ошибок, 4 варианта ответов (один правильный), правильные ответы с объяснениями и привязку к конкретному звонку для контекста обучения.

## Этап 9: Сохранение и доступность результатов

{TECHNICAL} В базе данных сохраняется оригинальная и обработанная транскрипция, оценки по каждому этапу чек-листа, результаты кастомной аналитики, данные о конверсиях и сгенерированный тест (при наличии ошибок).

{INTERFACE} Результаты появляются в интерфейсе сразу после обработки. Данные становятся доступны для анализа в разделах [Аналитика](#аналитика-коммуникации), [Тесты](#аналитика-тесты) и [Таблицы](#таблицы) согласно правам доступа пользователя.

## Этап 10: Интеграция и уведомления

{TECHNICAL} Результаты чек-листа автоматически отправляются в CRM через настроенные веб-хуки. Данные аналитики передаются по настроенным URL согласно конфигурации в разделе [Настройки - Подключение](#настройки-подключение).

Менеджеры видят новые оценки своих звонков в [разделе Коммуникации](#аналитика-коммуникации). Руководители получают уведомления о [тестах для утверждения](#аналитика-тесты). Администраторы контролируют общий процесс через статистику обработки.

## Временные характеристики обработки

{TECHNICAL} Полный цикл обработки звонка занимает 5-15 минут для стандартного звонка. Время зависит от длительности аудио и сложности настроенных анализов. Весь процесс полностью автоматизирован и не требует участия операторов.

### Время по этапам:
- Получение аудио: мгновенно
- Транскрибация: 2-5 минут  
- Разделение спикеров: 30 секунд
- Контроль качества: 30 секунд
- ИИ-анализ по чек-листам: 3-7 минут
- Кастомная аналитика: 1-3 минуты
- Генерация тестов: 1-2 минуты
- Сохранение и отправка: мгновенно
