# Процесс создания и управления тестами

## Автоматическое создание тестов

### Условия генерации тестов

**Когда система создает тест**:
- При наличии этапов с оценкой **-1 балл** (не выполнено)
- **Один тест на звонок** при обнаружении ошибок
- Тест генерируется автоматически сразу после анализа звонка

**Тесты. Общий вид.png**

### Принцип формирования тестов

**На основе ошибок**:
- ИИ анализирует этапы с отрицательной оценкой
- Выбирает наиболее критичные недостатки
- Формирует вопросы для закрепления правильного подхода

**Структура автоматически созданного теста**:
- Вопрос на основе допущенной ошибки
- Несколько вариантов ответа
- Правильный ответ с объяснением
- Связь с конкретным звонком и этапом

## Процесс редактирования тестов

### Кто может редактировать

**Права на редактирование имеют**:
- **Супер-администраторы**
- **Администраторы** 
- **Руководители офисов**

**Тесты. 1 - редактирование теста 2 - одобрить тест для отправки в аккаунт менеджеру.png**

### Что можно изменить

**Доступные для редактирования элементы**:
- **Формулировка вопроса** - корректировка для лучшего понимания
- **Варианты ответов** - изменение или добавление вариантов
- **Правильный ответ** - выбор наиболее подходящего варианта
- **Объяснение** - добавление пояснений к правильному ответу

**Тесты. Пример редактирование теста.png**

### Процесс редактирования

1. **Открытие теста** - переход в раздел "Аналитика → Тесты"
2. **Выбор теста** - нажатие на кнопку редактирования
3. **Внесение изменений** - корректировка содержания
4. **Сохранение** - фиксация изменений в системе

## Процесс утверждения тестов

### Статусы тестов

**Возможные статусы**:
- **Создан** - тест сгенерирован ИИ, ожидает утверждения
- **Утвержден** - тест готов для прохождения менеджером
- **Пройден** - менеджер выполнил тест
- **Отклонен** - тест требует доработки

**Тесты. Фильтр по статусу теста.png**

### Процесс утверждения

**Для руководителей офисов**:
1. **Фильтрация** - выбор тестов для менеджеров своего офиса
2. **Просмотр** - анализ содержания теста
3. **Редактирование** - корректировка при необходимости
4. **Утверждение** - подтверждение готовности к прохождению

**Тесты. Фильтр по менеджеру.png**

### Критерии для утверждения

**Руководитель проверяет**:
- Соответствие вопросов реальным ошибкам
- Корректность формулировок
- Адекватность вариантов ответов
- Образовательную ценность теста

## Отправка тестов менеджерам

### Автоматическая доставка

**После утверждения**:
- Тест автоматически становится доступен менеджеру
- Появляется в разделе "Аналитика → Тесты" у менеджера
- Отправляется уведомление о необходимости прохождения

### Привязка к звонку

**Связь с исходным звонком**:
- Тест прикреплен к конкретному звонку
- Менеджер может видеть, на основе какого звонка создан тест
- Возможность прослушать оригинальный звонок для контекста

## Прохождение тестов менеджерами

### Доступ к тестам

**Как менеджер находит тесты**:
1. Вход в систему через http://10.28.32.81/
2. Переход в раздел "Аналитика → Тесты"
3. Просмотр утвержденных тестов для прохождения

**Тесты. пример теста.png**

### Процесс прохождения

**Этапы выполнения теста**:
1. **Ознакомление** - чтение вопроса и вариантов ответа
2. **Выбор ответа** - выбор одного из предложенных вариантов
3. **Подтверждение** - фиксация выбранного ответа
4. **Получение результата** - просмотр правильного ответа и объяснения

### Результаты тестирования

**Что видит менеджер после прохождения**:
- Правильность выбранного ответа
- Объяснение правильного решения
- Рекомендации по улучшению
- Связь с исходным звонком для анализа

## Интеграция с обучением

### Связь с реальными звонками

**Образовательная ценность**:
- Тесты основаны на реальных ошибках
- Прямая связь с практической деятельностью
- Персональный подход к обучению каждого менеджера

### Повторное тестирование

**Возможности системы**:
- Создание новых тестов при повторных ошибках
- Отслеживание прогресса обучения
- Адаптация сложности под уровень менеджера

## Ограничения и особенности

### Один тест на звонок

**Принцип работы**:
- Даже при множественных ошибках создается только один тест
- Выбираются наиболее критичные недостатки
- Фокус на качественном, а не количественном обучении

### Обязательность прохождения

**Требования**:
- Утвержденные тесты обязательны для прохождения
- Результаты учитываются в общей статистике менеджера
- Влияют на оценку профессионального развития

**Важно**: Система тестирования направлена на практическое улучшение навыков менеджеров на основе их реальных ошибок в телефонных разговорах.
