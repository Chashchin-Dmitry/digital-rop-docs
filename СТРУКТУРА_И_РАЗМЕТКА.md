| 3.1 | Добавление этапа - название | `3.1 добавить этап - название.png` |

| 3.3 | Добавление этапа - когда игнорируем | `3.3 добавить этап - когда игнорируем.png` |
| 3.4 | Добавление этапа - порог оценки (0-100%) | `3.4добавить этап - после какого порога ставим +1 балл (от 0 до 100 процентов).png` |

**Текст из транскрипции (отредактированный):**

> **Шаблон скрипта** - это чек-лист, по которому ИИ оценивает звонки.
>
> При добавлении этапа в шаблон заполняются 4 поля:
>
> 1. **Название этапа** - краткое описание (например, "Приветствие")
>
> 2. **Критерии выполнения** - описание того, что должен сделать менеджер. Рекомендуем:
>    - Писать в безличной форме
>    - Использовать процентные веса для подэтапов
>    - Использовать буллет-поинты
>
> 3. **Когда игнорировать** - сценарии, когда этап не оценивается:
>    - Повторные звонки (клиент уже знаком с компанией)
>    - Короткие звонки
>    - Пишите обобщённо, в безличной форме
>
> 4. **Порог оценки (0-100%)** - минимальный процент для +1 балла:
>    - Рекомендуемое значение: 50-70%
>    - Если выполнение >= порога → +1 балл
>    - Если выполнение < порога → -1 балл
>    - Не рекомендуем ставить выше 70% - система становится слишком строгой

## Процесс обработки звонка (для FAQ/отдельного раздела)

1. **Получение звонка** - из CRM или телефонии
2. **Транскрибация** - аудио → текст
3. **Проверка конверсии** - да/нет по настроенному промту
4. **Оценка по чек-листу** - по привязанному шаблону скрипта
5. **Дополнительные промты** - извлечение кастомной информации
6. **Таблицы** - заполнение кастомных полей
7. **Сохранение результатов** - в базу данных
8. **Отправка в CRM** - комментарии, списки (если настроено)
9. **Формирование тестов** - в конце рабочего дня
